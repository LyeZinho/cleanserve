# SSL/TLS

CleanServe simplifies local development over HTTPS by providing automatic certificate generation and management.

## Auto HTTPS

The `SslManager` handles the creation and storage of self-signed certificates. These certificates are generated using the `rcgen` crate and stored in a consistent location on the local machine.

*   **Storage Path**: `~/.cleanserve/certs/`
*   **Format**: PEM format files (`.key` and `.crt`).
*   **Caching**: CleanServe reuses existing certificates if the `{domain}.key` and `{domain}.crt` files are present in the certificate directory.

## Certificate Specification

Every self-signed certificate generated by CleanServe follows these security standards:

*   **Type**: X.509
*   **Common Name (CN)**: The requested domain name.
*   **Organization (O)**: CleanServe
*   **Country (C)**: XX
*   **Validity**: 365 days.
*   **CA**: No (not a Certificate Authority).

## Protocol Security

CleanServe enforces modern security standards for all encrypted connections.

*   **Version**: TLS 1.3 exclusively.
*   **Legacy Support**: Support for TLS 1.0, 1.1, and 1.2 is disabled to prevent downgrade attacks and ensure the highest level of transport security.
*   **Configuration**: Protocol settings are managed by the `cleanserve-proxy` TLS configuration.
