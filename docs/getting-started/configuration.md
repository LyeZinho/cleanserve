# Configuration

CleanServe uses a `cleanserve.json` file to manage project settings. This file is automatically generated by `cleanserve init` and can be manually edited for advanced configuration.

## Configuration File

The `cleanserve.json` file is stored at your project root. Here is an example with all available configuration fields:

```json
{
  "name": "project-name",
  "engine": {
    "php": "8.4",
    "extensions": ["pdo_mysql", "gd", "intl"],
    "display_errors": true,
    "memory_limit": "256M"
  },
  "server": {
    "root": "public/",
    "port": 8080,
    "hot_reload": true
  }
}
```

## Configuration Schema

### Top-level Fields

- **`name`** (string): The name of your project. If initialized via `cleanserve init`, this is pulled from `composer.json`.

### Engine Settings

The `engine` object configures the PHP environment for your project.

- **`php`** (string, default: "8.4"): The PHP version for the server. CleanServe manages these versions automatically.
- **`extensions`** (array, default: []): A list of PHP extensions to enable. CleanServe ensures these are available for the selected PHP version.
- **`display_errors`** (boolean, default: true): Whether to show PHP errors on the page. Recommended for local development.
- **`memory_limit`** (string, default: null): The PHP memory limit (e.g., "128M", "512M"). If null, the default PHP limit is used.

### Server Settings

The `server` object configures the local development server behavior.

- **`root`** (string, default: "public/"): The document root for the server. Relative to the project root.
- **`port`** (integer, default: 8080): The primary server port.
- **`hot_reload`** (boolean, default: true): Enables Hot Module Replacement (HMR) for instant browser updates.

> **Note:** CleanServe uses `composer.json` for auto-detection during initialization, but once `cleanserve.json` is created, it becomes the source of truth for your development environment.
